Skill: Modern Fluid UI ArchitectDescription: Apply "Intrinsic Design" and modern CSS orchestration to all UI components to ensure they are high-fidelity, responsive, and robust.1. Layout & Sizing (The "Intrinsic" Rule)Avoid Fixed Pixels: Never use hardcoded width or height (e.g., 500px) unless for icons.Fluid Containers: Use min() for smart bounds: width: min(92%, 1200px);.Gap over Margin: Always use gap in Flex and Grid layouts. Never use margin-right or margin-bottom to space siblings.Smart Grids: Use repeat(auto-fit, minmax(min-content, 1fr)) for grids that reorganize themselves without media queries.2. Typography & Spacing (The "Clamp" Rule)Fluid Scaling: Use clamp() for font-size, padding, and margin to maintain visual rhythm across devices.Formula: clamp([min], [preferred_vw], [max]).Text Wrapping: Always apply text-wrap: balance to headings ($H1$ to $H3$) and text-wrap: pretty to body paragraphs to prevent orphans and widows.3. Visual Depth (The "Glass & Shadow" Rule)Glassmorphism: When using transparency, always pair bg-white/opacity with backdrop-blur-md and a subtle border-white/20.Layered Shadows: Avoid single-layer dark shadows. Use multiple transparent box-shadow steps to simulate realistic ambient occlusion.CSSbox-shadow: 0 1px 2px rgba(0,0,0,0.1), 0 4px 8px rgba(0,0,0,0.1), 0 12px 24px rgba(0,0,0,0.1);
4. Robustness (The "No-Cut-Off" Rule)Safe Overflow: Prefer overflow: clip for decorative elements. For content containers, always ensure padding accounts for potential scrollbars to prevent text-clipping.Min-Content Safety: Ensure cards have a min-height instead of a fixed height to accommodate variable text lengths without breaking the layout.5. Logic & StateCSS Variables: Centralize logic (spacing, colors, timing) in custom properties. Use Scoped Variables for component states (e.g., --card-bg: var(--bg-primary); changing to --card-bg: var(--bg-hover); on hover).
